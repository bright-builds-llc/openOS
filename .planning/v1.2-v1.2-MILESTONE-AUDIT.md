## Tech Debt Review

### Phase 18: App Submission Foundations

- Submitted manifest discovery is manual. Adding a new JSON file under `src/features/platform/submitted-apps/` is not enough by itself; the file must also be imported into `submittedAppManifests.ts`.
- `bun run submissions:check` validates only the imported manifest list, so an unregistered manifest file could be silently skipped by both validation and the catalog.

### Phase 17: Browser Direct Navigation

- Browser still has separate `browser-grid` and `browser` runtime entries with different storage namespaces. This is safe now because Browser stores no user state, but it is a risk if future Browser persistence/history is added without aliasing those entries.
- Grid-path Browser coverage is stronger than dock-entry coverage. This is low risk for the current milestone because both launcher entries resolve to the same Browser launch surface, but it remains follow-up debt.
